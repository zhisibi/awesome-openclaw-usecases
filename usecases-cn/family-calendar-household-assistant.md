现代一个普通家庭，往往有五六套日历：

- 自己的工作日历（公司账号 + 安全限制）
- 私人日历
- 家庭共享日历
- 孩子的学校日历（PDF、照片、网页）
- 各种兴趣班、夏令营的安排（邮件里、海报上）

除此之外，日常沟通还散落在 iMessage、微信、Telegram、邮件里：

- 牙医发来“明天 3 点见”的短信
- 老师群里发一张“本月活动安排”的图片
- 配偶问：“家里还有牛奶吗？”

这个用例的目标是：用 **OpenClaw 打造一个“家庭日历聚合 & 家务助理”**，让它：

- 每天早上自动汇总**全家人的日程**，发一份“早间简报”
- 在后台监控 iMessage 等信息流，自动把“隐性约定”变成日历事件
- 维护一个家庭库存（冰箱、储物间）数据库，让你随时问“还有没有 XX？”
- 帮你做购物清单、整理菜谱食材、规划出行时间

---

## 一、痛点：日历碎片化 + 家务信息分散

原文归纳了三个典型痛点：

1. **日历碎片化**
   - 工作日历不能随便共享
   - 学校活动在 PDF、图片、网页里
   - 夏令营安排躺在邮件附件
   - 结果就是：每天早上要一一打开各个 App / 网页检查

2. **家务协调成本高**
   - “牛奶还有吗？”——需要走到冰箱前看，然后再下楼去看储藏间
   - 一周下来这种“来回确认”消耗的精力非常大

3. **预约 / 活动容易漏**
   - 医院 / 牙医 / 银行确认短信发来，只是躺在对话列表里
   - 没有日历事件、没有提醒，最后变成“我忘了这回事”

这些事情都不是难事，但非常占认知带宽——是典型的 Agent 可以帮你“托管”的场景。

## 二、OpenClaw 家庭助理做了什么？

原文里的 workflow 可以拆成几块：

### 1. 早间日历简报（Morning Briefing）

每天早上固定时间（比如 8:00）：

- 读取所有家庭相关日历：
  - 工作日历（只读权限）
  - 家庭共享日历
  - 孩子学校日历（从 PDF / 图片 / 网页解析）
  - 最近几天新加的活动
- 汇总成一篇“**今日 + 未来 3 天日程总览**”
- 通过你指定的渠道发送：
  - Telegram 家庭群 / Slack channel / iMessage 群

### 2. iMessage / 短信中的“隐性日程”自动捕捉

在后台，Agent 会定期扫描最近的 iMessage / 短信记录：

- 识别类似：
  - “Your appointment is confirmed for…”
  - “我们周五晚上一起吃饭吧？”
  - “Practice moved to Saturday at 3pm”
- 一旦识别到“这是个约定 / 事件”：
  - 自动创建对应的日历事件（标题、时间、地点）
  - 在日历中自动加入**行程前后缓冲时间**（例如提前 30 分钟出发，结束后预留 30 分钟）
  - 在家庭群里发一条确认消息：

> “已为你创建牙医预约：周二 2 点，已添加 1:30–2:00、3:00–3:30 的交通时间块。”

### 3. 家庭库存（Pantry & Fridge Inventory）

Agent 会维护一个本地的 `inventory.json` 或数据库：

- 记录每个物品：名称、数量、存放位置（冰箱 / 厨房 / 地下室）、最近更新时间、低库存阈值
- 更新方式可以是：
  - 文字：“我们没有鸡蛋了” -> 更新鸡蛋库存为 0
  - 照片：拍一张冰箱 / 冰柜 / 储物间照片 -> 用视觉模型识别物品和数量
  - 小票：拍超市小票 -> 根据商品和数量更新库存

你和家人可以随时在聊天里问：

- “我们还有黄油吗？”
- “列出所有快要用完的东西。”
- “基于现有食材，推荐三道菜。”

### 4. 购物清单与菜谱整合

在库存数据基础上，Agent 可以：

- 维护一个“本周计划做的菜 / 菜谱清单”
- 从菜谱中抽取食材需求，与现有库存比对：
  - 生成“缺口清单”（Shopping List）
  - 自动作去重、合并相同食材

当你要去超市时，只需让 Agent 导出最新购物清单即可。

## 三、需要的技能与环境

原用例依赖的能力主要有：

1. 日历 API：
   - Google Calendar API
   - Apple Calendar / iCloud（通过 CalDAV 或 EventKit）

2. iMessage Skill（macOS 环境）
   - OpenClaw 社区已有 iMessage 集成案例
   - 建议在家里放一台常开的 Mac Mini，既跑 OpenClaw，又接 iMessage

3. 家庭聊天入口
   - Telegram 群 / Slack 频道 / iMessage 群聊

4. 本地文件系统
   - 用于存储 `inventory.json`、OCR 后的日历结构化数据、收据记录等

5. 图像识别 / OCR
   - 将照片中的学校日历、活动海报、冰箱内容转成可解析的数据

## 四、在 OpenClaw 中可以怎么落地？

你可以把这个“家庭助理”拆成多个 HEARTBEAT 任务：

### 1. 每日早间简报

```text
每天 08:00：
1. 读取：我的工作日历（只读）、家庭共享日历、配偶的共享日历；
2. 解析 ~/Documents/school-calendars/ 目录下新增的 PDF / 图片，更新学校日历事件；
3. 汇总今天 + 未来 3 天全家的活动安排，包括冲突提示；
4. 把简报发送到我们的 Telegram 家庭群。
```

### 2. 每 15 分钟的消息监听

```text
每 15 分钟：
1. 检查最近的 iMessage / 短信；
2. 识别其中的预约确认、约会安排、练习时间变更等“事件类”信息；
3. 为每个事件创建日历记录，并自动添加往返路程缓冲时间；
4. 在家庭群里发一条说明：「我帮你把这个新约会记到日历里了」。
```

### 3. 家庭库存与购物清单

```text
随时响应这些指令：
- “更新库存：我们没有鸡蛋了。”
- “从这张收据 / 照片更新库存。”
- “列出所有低于阈值的物品，生成购物清单。”
- “根据本周计划做的菜，补充需要购买的食材。”
```

## 五、关键经验 & 小技巧

原文和社区讨论里，有几个值得借鉴的点：

1. **先从只读开始**
   - 先让 Agent 只“读日历、读消息、给建议”，不直接写日历
   - 等你对它的判断比较信任以后，再放开“自动创建事件”的权限

2. **用家庭群做“操作透明化”**
   - 凡是自动创建 / 修改的事件，都在家庭群里发一条说明
   - 这样一来出错很容易被发现，也帮助家人建立信任

3. **照片输入很实用**
   - 很多学校 / 兴趣班只给一张 PDF / 照片，OCR+解析比手抄省太多时间
   - 同理，冰箱 / 冷柜拍照比逐条登记库存高效得多

4. **Mac Mini 是理想的宿主机**
   - 既连接 iMessage，又随时在线
   - 还能跑本地的自动化、备份任务

## 六、适合谁用？

这套“家庭日历 + 家务助理”特别适合：

- 有小孩的家庭：学校活动、兴趣班、假期安排很多、信息源复杂
- 双职工家庭：每天的时间已经被工作掏空，不想再被琐事占用脑力
- 喜欢用数字工具管理生活的人：愿意花一点时间搭一套系统，让以后每天都省一点

它不是一个“新日历 App”，而是：

> 用 OpenClaw 把你现有的日历 / 消息 / 照片 / 文件串在一起，做一个真正懂你家庭节奏的“生活中枢”。
